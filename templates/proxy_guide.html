Why intercept traffic?

Modern web applications perform many checks on the client-side (in your browser) to prevent invalid input. By intercepting the raw HTTP request after it leaves the browser but before it reaches the server, you can bypass browser restrictions, unhide hidden form fields, and manipulate cookies.

Setup Burp Suite Community Edition

  1. Download and Install: Get Burp Suite Community Edition from PortSwigger.
  2. Start Burp Suite: Open the application, choose "Temporary Project," and click "Start Burp".
  3. Configure Proxy: Go to the Proxy tab, then the Proxy settings side-tab. Ensure the interface is listening on 127.0.0.1:8080 and the box is checked.
  4. Use Burp's Built-in Browser (Easiest): Go back to the Intercept tab and click Open Browser. This launches a pre-configured Chromium browser that automatically routes traffic through Burp.
  5. Navigate to Your Lab: In that new browser, go to {% if container %}http://localhost:{{ container.port }}{% else %}your lab url{% endif %}.
  6. Intercept! Turn "Intercept is on" in the Proxy tab. Try logging in. The request will freeze in your browser and appear in Burp Suite, allowing you to edit the raw traffic before clicking "Forward".
